# Chompo — Интерпретатор Py-like языка на C++

Проект представляет собой интерпретатор динамически типизированного языка программирования **Chompo** (файлы с расширением `.chmp`), написанный на современном C++23. Язык Chompo сочетает в себе синтаксические черты Python (динамическая типизация, простота) и C++/JavaScript (блочная область видимости, фигурные скобки, явное объявление переменных).

## 🛠️ Архитектура и компоненты

### 1. Лексический анализатор (lexer)
Сканирует исходный код символ за символом и генерирует поток токенов.
* `token.h` определяет перечисление `TokenType` (ключевые слова `var`, `fun`, `if`, `else`, `while`, `print`, литералы, операторы и т.д.) и структуру `Token`, хранящую позицию в кодовой базе (строка, колонка).
* `lexer.cpp` выполняет распознавание регулярных структур (числа, строки, идентификаторы) и ключевых слов.

### 2. Синтаксический анализатор (parser)
Построен по схеме **Pratt Parser** (парсер с таблицей приоритетов операторов). Превращает плоский список токенов в иерархическое дерево AST.
* `ast.h` использует `std::variant` и `std::unique_ptr` для эффективного и безопасного представления узлов выражений (`Expr`) и инструкций (`Stmt`).
* `parser.cpp` реализует разбор на основе приоритетов (Precedence table), что упрощает обработку бинарных и унарных операций.

### 3. Интерпретатор (interpreter)
Выполняет обход AST (Tree-walk interpreter).
* `value.h`: Класс `Value` представляет рантайм-значение Chompo на базе `std::variant` (`std::monostate` для `NULL`, `bool`, `std::int64_t`, `std::string`, `ArrayPtr`).
* `environment.h`: Класс `Environment` хранит переменные текущей области видимости и ссылку на родительское (внешнее) окружение для поддержки вложенности блоков.
* `interpreter.cpp`: Вычисляет значения выражений и выполняет инструкции, выводя результаты (например, для функции `print`).

## 🚀 Текущие возможности языка

На данный момент интерпретатор Chompo успешно поддерживает:
* **Объявление переменных** с блочной областью видимости:
  ```javascript
  var result = 0;
  {
      var inner = 42;
      print(inner);
  }
  // inner здесь недоступен
  ```
* **Массивы**: создание через литерал фигурных скобок при объявлении:
  ```javascript
  var numbers[] = {1, 2 + 3, 4};
  ```
* **Управляющие конструкции**:
  * Ветвления `if` - `else`.
* **Базовые типы**: целые числа (`integer`), строки (`string`), логический тип (`bool`), списки (`array`) и `NULL`.
* **Операции**:
  * Арифметические: `+`, `-`, `*`, `/`, `%`.
  * Сравнения: `==`, `!=`, `<`, `<=`, `>`, `>=`.
  * Логические: `&&`, `||`, `!`.
  * Конкатенация строк (для двух операндов-строк).

## 📦 Сборка и запуск

Для сборки проекта вам понадобятся компилятор с поддержкой C++23 (например, GCC 13+ или Clang 16+), CMake 4.2+ и система сборки (Ninja или Make).

```bash
# 1. Настройка сборки
cmake -B build

# 2. Компиляция
cmake --build build

# 3. Запуск интерпретатора с тестовым файлом
./build/Chompo tests/test_code.chmp
```

## 🗺️ План реализации (Roadmap)

Ниже представлен детальный план расширения синтаксиса и возможностей Chompo на ближайшее будущее:

### 1. Массивы: явное создание и операции с элементами
- **Явный конструктор массивов**:
  Добавить поддержку явного синтаксиса создания массивов вида:
  ```javascript
  var arr = Array{1, 2, 3};
  ```
  Для этого потребуется добавить ключевое слово `Array` в список токенов и поддержать его разбор в парсере.
- **Доступ к элементам и мутация (Subscript Access & Mutation)**:
  Реализовать операции чтения и записи элементов массива по индексу:
  ```javascript
  var x = arr[0];
  arr[1] = 10;
  ```
  Необходимо добавить новый тип выражения в AST (например, `SubscriptExpr { ExprPtr target; ExprPtr index; }`) и поддержать его в интерпретаторе с проверкой выхода за границы.

### 2. Преобразование типов (Type Conversion)
- **Явное преобразование**:
  Внедрить встроенные функции-конструкторы для приведения типов:
  * `int(value)` — преобразование в целое число (из строки или bool).
  * `str(value)` — преобразование любого значения в строковое представление.
  * `bool(value)` — приведение значения к логическому типу.
- **Неявное преобразование (Coercion)**:
  Определить правила неявного приведения типов при вычислениях. Например:
  * Использование не-логических значений в условиях `if` (приведение к `bool` через проверку `is_truthy`).
  * Сложение строки с числом (автоматическое приведение числа к строке).

### 3. Строки как массивы символов
- **Конкатенация строк**:
  Обеспечить более гибкое сложение строк, в том числе с поддержкой неявного приведения других типов к строкам при сложении (`"age: " + 18`).
- **Индексация строк**:
  Добавить возможность читать отдельные символы строки по индексу:
  ```javascript
  var s = "hello";
  var c = s[1]; // 'e'
  ```
  Строка будет вести себя аналогично массиву символов при чтении.

### 4. Поддержка символьного типа (Character Support)
- **Литералы символов**:
  Добавить поддержку одиночных кавычек в лексере для представления символов:
  ```javascript
  var ch = 'a';
  ```
- **Интеграция в рантайм**:
  Добавить тип `char` (или представить его как односимвольную строку / целое число в `Value`) и соответствующие токены в лексере.

### 5. Имена переменных с символом подчеркивания
- Поддержка `_` в идентификаторах на любых позициях (например, `my_cool_variable`, `_temp`).
  * *Примечание:* лексер уже позволяет распознавать `_` через метод `Lexer::is_alpha`, но необходимо обеспечить корректную валидацию имен переменных и добавить тестовые покрытия.

### 6. Полная поддержка функций
- **Объявление функций**:
  Поддержать синтаксис определения именованных функций:
  ```javascript
  fun add(a, b) {
      return a + b;
  }
  ```
  Необходимо реализовать `FunStmt` (или `FunctionExpr`), содержащий список параметров и тело (блок инструкций).
- **Функции как объекты первого класса (First-Class Functions)**:
  Предоставить возможность сохранять функции в переменные и передавать их в качестве аргументов:
  ```javascript
  var my_func = add;
  var result = my_func(2, 3);
  ```
  Для этого рантайм-хранилище `Value` должно быть расширено типом callable (объект функции / замыкания).
- **Активация вызовов (CallExpr)**:
  Дописать в интерпретаторе поддержку `evaluate_node(const CallExpr&)`, которая создает новое окружение выполнения (`Environment`), привязывает переданные аргументы к параметрам функции и выполняет её тело.

### 7. Расширение набора токенов
Добавить новые токены в `token.h` и `lexer.cpp` для обслуживания новых возможностей:
- Токены приведения типов.
- Токен одиночной кавычки и символов.
- Дополнительные операторы (например, оператор диапазона или инкременты/декременты).

### 8. Доработка интерпретатора (Control Flow & Loops)
- Реализовать выполнение циклов `while`, `for`, а также конструкций итерации по коллекциям `for element in array`.
- Добавить поддержку операторов управления потоком выполнения: `break`, `continue` и `return`. Это потребует внедрения механизма исключений управления (Control Flow Exceptions) в обходчик дерева интерпретатора.
